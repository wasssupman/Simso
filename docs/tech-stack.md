# 심소(SIMSO) — 기술 스택

---

## 아키텍처 개요

```
┌───────────────────────────────────────────┐
│                클라이언트                    │
│                                           │
│  웹 브라우저 ◄─── Next.js (SSR) ──────────┤
│                       ▲                   │
│  모바일 앱 ◄── React Native WebView ──────┤
│                 (같은 도메인 로드)            │
└─────────────────────┬─────────────────────┘
                      │
                      ▼
┌───────────────────────────────────────────┐
│          Next.js API Routes               │
└────┬──────────┬──────────┬────────────────┘
     │          │          │
     ▼          ▼          ▼
 PostgreSQL   Redis     S3 (이미지)
             (캐시)
     │
     └──── AI 파이프라인 (업로드 전 검사) ────┐
           │                               │
           ├─ Claude API (금지 소재/논조)     │
           ├─ AI 생성물 탐지 API             │
           └─ 이미지 안전성 검사              │
                                           ▼
                                      게시물 공개
```

**핵심 전략**: Next.js 단일 코드베이스로 웹·앱 동시 커버.
앱은 같은 도메인을 WebView로 로드하므로, 배포 한 번이면 양쪽 반영.

## 프론트엔드 (웹)

| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | **Next.js 15 (App Router)** | SSR + API Routes 통합 |
| 언어 | **TypeScript** | 타입 안전성 |
| 스타일링 | **Tailwind CSS** | 뉴트럴 톤 디자인 시스템 빠르게 구축 |
| 상태 관리 | **Zustand** | 가볍고 심플 |
| 데이터 페칭 | **TanStack Query** | 피드 캐싱, 낙관적 업데이트 |
| 폼 | **React Hook Form + Zod** | 300자 제한 등 유효성 검사 |

## 모바일 앱 (WebView 래핑)

| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | **React Native (Expo)** | WebView 래핑 + 네이티브 브릿지 |
| WebView | **react-native-webview** | `https://simso.app` 그대로 로드 |

앱에서 네이티브로 처리할 것:

| 네이티브 기능 | 방식 |
|--------------|------|
| 푸시 알림 | FCM (Android) / APNs (iOS) |
| 카메라 | 네이티브 브릿지 → 웹에 이미지 전달 |
| 공유 | 네이티브 공유 시트 |
| 오프라인 | 스플래시 + 오프라인 안내 화면 |

그 외 모든 UI는 WebView 내 Next.js가 담당.

## 백엔드

| 항목 | 선택 | 이유 |
|------|------|------|
| API | **Next.js Route Handlers** | 별도 서버 없이 시작 |

| 인증 | **Auth.js v5** | 이메일 로그인 기반, 소셜 최소화 |

## AI 파이프라인

게시물 업로드 시 순차 실행되는 검사 단계:

```
사용자 글 작성 → [업로드 요청]
  │
  ▼
① 금지 소재 검사 ─── 정치·종교·혐오·광고 탐지
  │                  → 해당 시 차단 + 사유 안내
  ▼
② AI 생성물 탐지 ─── AI가 쓴 글/이미지 판별
  │                  → 해당 시 차단
  ▼
③ 논조 분석 ──────── 부정적·공격적 톤 감지
  │                  → 차단 아님. 넛지 문구 표시
  │                    "다른 분의 감정도 생각해주세요"
  │                  → 사용자가 수정 또는 그대로 게시 선택
  ▼
④ 이미지 안전성 ──── 부적절 이미지 필터링
  │                  → 해당 시 차단
  ▼
게시물 공개
```

| 단계 | 기술 | 비고 |
|------|------|------|
| ① 금지 소재 | **Claude API** | 프롬프트 분류. 한국어 이해도 높음 |
| ② AI 텍스트 탐지 | **Claude API 또는 GPTZero** | 초기 API, 데이터 쌓이면 자체 모델 검토 |
| ② AI 이미지 탐지 | **Hive Moderation** | AI 생성 이미지 탐지 특화 |
| ③ 논조 분석 | **Claude API** | 넛지 판단용 (차단 아닌 안내) |
| ④ 이미지 안전성 | **AWS Rekognition** | 성인·폭력 콘텐츠 탐지 |

비용 참고: ①③은 하나의 Claude API 호출로 묶을 수 있음 (금지 소재 + 논조를 한 번에 판단).

## 피드 알고리즘

concept.md 기준: **관심 키워드 60% + 다양한 관심사 40%**

```
피드 조회 요청
  │
  ├─ 60% ── 사용자 관심 키워드와 매칭되는 게시물 (최신순)
  │
  └─ 40% ── 관심사 밖 게시물 (랜덤 샘플링, 다양성 확보)
```

- 온보딩 시 관심 키워드 선택 (산책, 요리, 독서, 비움 등)
- 게시물 작성 시 키워드 태그 선택 (또는 AI 자동 태깅)
- 추천 점수 랭킹 없음 — 시간순 기반 혼합

| 항목 | 선택 | 이유 |
|------|------|------|

## MVP 개발 순서

concept.md에서 명시: **AI 검사가 콘텐츠 기능보다 먼저**.

```
Phase 0 — 프로젝트 셋업
  모노레포 초기화, DB 스키마, 인증

Phase 1 — AI 파이프라인 (최우선)
  금지 소재 검사 API
  AI 생성 콘텐츠 탐지 API
  논조 분석 + 넛지 API
  이미지 안전성 검사

Phase 2 — 핵심 기능
  오늘의 소소함 (1일 1글 + AI 검사 연동)
  메인 피드 (60/40 알고리즘)
  공감해요 리액션
  프로필

Phase 3 — 확장
  계절 게시판 (비움 일지)
  React Native WebView 앱
  푸시 알림

Phase 4 — 커뮤니티
  편지함
  조용한 챌린지
```
